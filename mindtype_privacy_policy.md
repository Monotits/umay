## 1. Overview

MindType is a game theory-based decision-making app that helps you understand your thinking patterns through short micro-games. We are committed to protecting your privacy. This Privacy Policy explains what data we collect, how we use it, and your rights.

**Key principle:** MindType uses Firebase Anonymous Authentication — you do not need to provide any personal information (no email, no name, no password) to use the app. Your identity is represented by a random anonymous identifier.

---

## 2. Data We Collect

### 2.1 Data Stored in Cloud Firestore (Firebase)

The following data is stored securely in Google Cloud Firestore, associated with your anonymous identifier:

| Data | Purpose | Storage |
|------|---------|---------|
| Anonymous user ID (Firebase Auth) | Account identification | Cloud Firestore |
| Game results (choices, scores, time taken) | Archetype calculation and stats | Cloud Firestore |
| Cognitive bias patterns | Personal profile generation | Cloud Firestore |
| Archetype profile | Your decision-making type | Cloud Firestore |
| Daily streak data | Motivation and streak tracking | Cloud Firestore |
| AI-generated insights | Personalized feedback (Gemini) | Cloud Firestore |
| Weekly/monthly reports | Personalized analysis (Gemini) | Cloud Firestore |
| Subscription status | Feature access control | Cloud Firestore + RevenueCat |
| Notification preferences | Reminder timing | Cloud Firestore |
| Selected language | UI language preference | Cloud Firestore |
| FCM token | Push notification delivery | Cloud Firestore |

**Important:** None of this data is linked to your real identity. Your anonymous ID is a random string generated by Firebase — it cannot be traced back to you unless you voluntarily link an Apple or Google account.

### 2.2 Data Stored On Your Device (Local Cache)

| Data | Purpose | Storage |
|------|---------|---------|
| Cached user profile | Offline access | UserDefaults |
| Daily game counter | Rate limiting | UserDefaults |
| Onboarding state | App initialization | UserDefaults |
| Language preference | Quick access | UserDefaults |
| Subscription status cache | Offline access | UserDefaults |

### 2.3 Data Collected by Third-Party Services

#### Firebase Authentication (Google LLC)
- We use Firebase Anonymous Authentication. No personal information is required.
- If you choose to link an Apple or Google account, the linked account email is stored by Firebase Auth only — we do not access or store it in our database.
- **Privacy policy:** https://policies.google.com/privacy

#### Cloud Firestore (Google LLC)
- Game data and profiles are stored in Google Cloud Firestore servers.
- Data is encrypted at rest (AES-256) and in transit (TLS 1.3).
- **Data location:** Google Cloud (region configured by developer)
- **Privacy policy:** https://policies.google.com/privacy

#### Google Gemini AI (via Vertex AI / Cloud Functions)
- We use Google Gemini Flash to generate personalized game scenarios, insights, and reports.
- **What is sent to Gemini:** Aggregated game statistics and archetype data (no personal identifiers, no raw game choices).
- **What Gemini produces:** Game scenarios, weekly/monthly reports, and personal insights in your selected language.
- All AI-generated content is cached in Firestore and not regenerated.
- **Privacy policy:** https://policies.google.com/privacy

#### Firebase Analytics (Google LLC)
- Anonymous usage events (screen views, game completion rates).
- **What is sent:** Anonymous usage events. No personal identifiers, no game choices, no archetype data.
- **Privacy policy:** https://policies.google.com/privacy

#### Firebase Crashlytics (Google LLC)
- Crash reports: device model, OS version, app version, stack trace. No personal data.
- **Privacy policy:** https://policies.google.com/privacy

#### Firebase Cloud Messaging (Google LLC)
- Push notifications for insights, reports, and reminders.
- **What is stored:** Anonymous FCM device token. Not linked to personal identity.
- **Privacy policy:** https://policies.google.com/privacy

#### RevenueCat
- Manages subscription state.
- **What is sent:** Anonymous device identifier (= Firebase UID), purchase transaction data from Apple.
- **Privacy policy:** https://www.revenuecat.com/privacy

#### Apple App Store / StoreKit
- Subscription purchases are processed entirely through Apple's App Store. We never receive your payment details.
- **Privacy policy:** https://www.apple.com/legal/privacy/

### 2.4 Data Collected via Contact Support Form

If you choose to use the in-app Contact Support form, the following data is submitted:

| Data | Purpose | Storage |
|------|---------|---------|
| Survey answers (app rating, scenario quality, favorite feature) | Product feedback | Cloud Firestore |
| Your message text | Support request | Cloud Firestore |
| Email address (optional, only if you provide it) | Response communication | Cloud Firestore |
| App version, locale, subscription status | Debugging context | Cloud Firestore |

This data is voluntarily submitted and associated with your anonymous user ID. You are rate-limited to one submission per 24 hours.

### 2.5 Data We Do NOT Collect

- No email, name, or password required — anonymous auth by default
- No location data
- No contacts or address book access
- No camera or microphone access
- No advertising identifiers (IDFA) — we do not use ad networks
- No biometric data
- No financial or payment data (all purchases go through Apple)
- No browsing history or cross-app tracking

| Data | How We Use It |

---

## 4. AI-Generated Content

MindType uses Google Gemini Flash AI to generate:
- Game scenarios (daily batch, cached permanently)
- Personal insights when your profile changes
- Weekly and monthly decision analysis reports (Premium)
- Weakness development recommendations (Premium)

**What is shared with Gemini:** Only aggregated, anonymized game statistics (e.g., "user scored 72% on cooperation across 15 games"). No raw game choices, no personal identifiers, and no device information is sent to the AI model.

---

## 5. Push Notifications

MindType uses **Firebase Cloud Messaging (FCM)** for:
- **Surprise insights:** Server-triggered notifications when your profile changes
- **Report notifications:** When weekly/monthly reports are ready
- **Daily reminders:** Scheduled locally on your device

Notifications require your explicit permission, which you can revoke at any time in iOS Settings > Notifications > MindType.

## 6. Data Retention

| Data | Retention |
|------|-----------|
| Firestore user data | Until you request deletion |
| AI-generated content | Until you request deletion |
| Firebase Analytics data | 14 months (Firebase default) |
| Crashlytics crash reports | 90 days (Firebase default) |

**Deleting your data:** Go to MindType Settings → "Delete All Data" to permanently remove all your data from our servers (Firestore, Firebase Auth, all associated records). This action is irreversible and processed by a secure Cloud Function.

## 7. Children's Privacy

MindType is intended for users aged **13 and older**. We do not knowingly collect personal information from children under 13. If you believe a child under 13 has used the app, please contact us at support@mindtype.app.

## 8. Your Rights

- **Access:** View your data directly through the app (Stats, Profile screens).
- **Export:** Go to Settings → "Export Data" to download a JSON copy of your complete data.
- **Delete:** Go to Settings → "Delete All Data" to permanently remove all data from our servers.
- **Opt out of Analytics:** Limit ad tracking in iOS Settings → Privacy → Apple Advertising.
- **Account Linking:** You choose whether to link an Apple/Google account. Unlinking is available in Settings.
- **Notification Control:** Enable/disable notifications at any time in iOS Settings.

For questions or requests regarding your data, contact us at support@mindtype.app.

---

## 9. Data Security

- All data in transit is encrypted with TLS 1.3
- Firestore data at rest is encrypted with AES-256 (Google-managed keys)
- Local cached data is protected by iOS file system encryption
- Firebase Auth tokens are auto-refreshed and securely managed
- Cloud Functions run in Google's secure environment
- No sensitive data in URL parameters or query strings
- Firestore security rules enforce user-level data isolation
- RevenueCat webhook signature verification for subscription events
